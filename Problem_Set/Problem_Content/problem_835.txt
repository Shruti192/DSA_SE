b'\n\xc2\xa0\nProblem description.\nYou are given two sequences of N elements P and Q. You have to partition them where the cost of making a partition from index L to R is cost = min(P[L...N])*max(Q[1...R]). Find the minimum and the maximum cost of partitioning the two sequences into exactly K partitions. Remember, the partitions of the two sequences must be the same, you cannot make different partitions for P and Q.\nInput\nInput description.\nThe first line of the input contains an integer T denoting the number of test cases. T test cases follow. The first line of each test case contains a two integers N and K denoting the number of elements in the sequences and the number of partitions to be made. Then follows N lines, each line containing two integers, the first element corresponding the P and the second integer corresponding to Q.\n\nOutput\nFor each test case, output a single line containing two integers denoting minimum and maximum cost.\nConstraints\n\n\n1 \xe2\x89\xa4 T \xe2\x89\xa4 10\n1 \xe2\x89\xa4 K \xe2\x89\xa4 N \xe2\x89\xa4 1000\n-105 \xe2\x89\xa4 Pi, Qi \xe2\x89\xa4 105\n\n\nSubtasks\n\nSubtask #1(9 points):\n\n1 \xe2\x89\xa4 N \xe2\x89\xa4 1000\nK = 2\n\nSubtask #2(16 points):\n\n1 \xe2\x89\xa4 K \xe2\x89\xa4 N \xe2\x89\xa4 1000\n1 \xe2\x89\xa4 Pi, Qi \xe2\x89\xa4 105\n\nSubtask #3(27 points):\n\n1 \xe2\x89\xa4 K \xe2\x89\xa4 N \xe2\x89\xa4 100\n\nSubtask #4(48 points):\n\nOriginal Constraints\n\n\nExample\nInput:\n3\n5 1\n1 1\n2 2\n3 3\n4 4\n5 5\n\n5 5\n1 1\n2 2\n3 3\n4 4\n5 5\n\n5 2\n1 1\n2 2\n3 3\n4 4\n5 5\n\nOutput:\n5 5\n55 55\n11 29\n\n\n\n\n\nExplanation\n\nExample case 1.\nThere is only one way to partition the elements and that is to keep everything in the same partition, i.e, [1, 5]. The cost is min(P[1..N])*max(Q[1..N]) = 1*5 = 5.\nThis is both the maximum and the minimum possible.\n\n\n\n\nAuthor:\n3\xe2\x98\x85ista2000\n\n\nEditorial:\nhttps://discuss.codechef.com/problems/MINMAX2\n\n\nTags:\n\n               \n                 convex-hull, dynamic-programming, icop1806, ista2000, medium-hard\n                 \n               \n             \n             \n\n\n\n                        Difficulty Rating:\n                    \n\nN/A\n\n\nDate Added:\n2-01-2018\n\n\nTime Limit:\n1 secs\n\n\nSource Limit:\n50000 Bytes\n\n\nLanguages:\nPYTH 3.6, JAVA, C, CPP14, PYTH, PYP3, CS2, ADA, PYPY, TEXT, PAS fpc, NODEJS, RUBY, PHP, GO, HASK, TCL, kotlin, PERL, SCALA, LUA, BASH, JS, rust, LISP sbcl, PAS gpc, BF, CLOJ, D, CAML, swift, FORT, ASM, FS, WSPC, LISP clisp, SCM guile, PERL6, ERL, CLPS, PRLG, ICK, NICE, ICON, COB, SCM chicken, PIKE, SCM qobi, ST, NEM\n\n\n\n\n\nSubmit\n\n'
